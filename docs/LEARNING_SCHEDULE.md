# 12 周学习时间表

## 学习设定

- 学习周期：12 周
- 每日投入：4-6 小时
- 学习目标：3 个月内具备投递 AI Agent 应用开发实习的基础能力
- 学习策略：边学边做，边做边投，不走纯理论路线

## 12 周学习总表

| 周数 | 学习重点 | 具体内容 | 本周产出 |
| --- | --- | --- | --- |
| 第 1 周 | Python + Web 基础 | Python 复习、环境配置、Git、Linux 基础、HTTP/JSON、FastAPI 入门 | 跑通一个 FastAPI Hello World 项目，学会 Git 提交 |
| 第 2 周 | 模型 API 调用 | 大模型 API 调用、Prompt 基础、流式输出、环境变量管理、异常处理 | 做一个文本总结/改写 API |
| 第 3 周 | 结构化输出 | JSON Schema、Pydantic、structured outputs、信息抽取 | 做一个岗位描述/简历信息抽取器 |
| 第 4 周 | 工具调用入门 | function calling、工具 schema、工具结果回传、简单对话状态 | 做一个天气/搜索/日历模拟工具调用 Agent Demo |
| 第 5 周 | RAG 基础 | 文档切分、embedding、向量检索、top-k、相似度检索 | 做一个最小版文档问答 Demo |
| 第 6 周 | RAG 优化 | chunk 策略、引用来源、未命中兜底、基础 rerank 思路 | 完成可演示的文档问答 Agent |
| 第 7 周 | Agent workflow | Router、Planner/Executor、状态管理、失败重试 | 把 Agent 改成多步骤执行 |
| 第 8 周 | 前后端联调 | React/Next.js 基础或简单页面、聊天 UI、文件上传、接口联调 | 给 RAG Agent 加一个可演示前端 |
| 第 9 周 | 业务场景设计 | 选择主项目方向：简历筛选 / SQL Agent / 客服工单分流 / 会议纪要 | 完成主项目需求文档 |
| 第 10 周 | 主项目开发 | Agent 逻辑、工具接入、状态管理、权限控制、异常处理 | 主项目核心功能完成 |
| 第 11 周 | 工程化与部署 | Docker、日志、README、接口文档、简单评测、部署到云平台或本地容器 | 主项目可演示、可部署 |
| 第 12 周 | 求职准备 | 简历优化、GitHub 整理、项目介绍稿、常见面试题、开始投递 | 完整简历 + 项目作品集 + 开始投实习 |

## 按阶段拆分

### 第 1 个月：先把 AI 应用跑起来

目标：

- 能独立写一个 FastAPI 服务
- 能把模型接入后端
- 能让模型稳定返回 JSON
- 能做简单工具调用

阶段项目：

- 基础工具调用 Agent

### 第 2 个月：重点做 RAG 和 Agent workflow

目标：

- 会做完整 RAG
- 会做带引用的问答
- 会做基础 workflow
- 会把前后端联调跑通

阶段项目：

- RAG 文档问答 Agent

### 第 3 个月：做业务项目并开始投递

目标：

- 完成一个业务型 Agent
- 具备基本工程化与部署能力
- 完成简历、GitHub 和项目介绍材料
- 开始真实投递实习

阶段项目：

- 业务型 Agent 主项目

## 每天学习安排

建议每天按下面的节奏推进：

| 时间段 | 任务安排 |
| --- | --- |
| 前 2 小时 | 学新内容：看文档、看教程、跑官方 Demo、理解概念 |
| 中间 2 小时 | 动手做：写接口、写工具、调试 Bug、联调前后端 |
| 最后 1-2 小时 | 整理输出：提交 GitHub、写 README、记录问题、做复盘 |

## 每周固定任务

每周都必须完成以下 4 件事：

1. 至少提交 4-5 次 Git
2. 写项目 README 或学习记录
3. 把本周学习内容做成一个小功能
4. 周末做一次复盘

## 每月验收标准

### 第 1 个月验收

你应该能清楚回答：

- 怎么调用模型 API
- 什么是 structured outputs
- function calling 是怎么工作的
- FastAPI 怎么写一个接口

### 第 2 个月验收

你应该能清楚回答：

- RAG 为什么需要 chunk
- 怎么做文档切分
- 为什么会检索不到
- 怎么给回答加引用来源
- Agent workflow 和普通聊天接口有什么区别

### 第 3 个月验收

你应该能清楚回答：

- 主项目解决什么问题
- 为什么这样设计工具和流程
- 怎么控制错误调用
- 怎么评估效果
- 怎么部署和演示

## 最终项目组合建议

### 项目 1：基础工具调用 Agent

适合展示：

- function calling
- structured outputs
- 工具调度能力

### 项目 2：RAG 文档问答 Agent

适合展示：

- 文档处理与检索
- 回答引用来源
- 前后端联动

### 项目 3：业务型 Agent

推荐优先级：

1. SQL 数据分析 Agent
2. 简历筛选 Agent
3. 客服工单分流 Agent
4. 会议纪要 Agent

## 第 12 周前必须准备好的求职材料

1. 简历
2. GitHub 项目仓库
3. 每个项目的 README
4. 每个项目的 1 分钟介绍稿
5. 每个项目的 3 分钟介绍稿
6. 常见面试题准备

## 投递节奏建议

- 第 8 周：开始整理简历
- 第 9 周：开始投第一批
- 第 10-12 周：边投边改项目
- 第 12 周：集中准备面试

## 学习中要避免的坑

1. 只看概念，不做项目
2. 一开始就学多 Agent
3. 同时浅尝太多框架
4. 项目做得过于花哨却缺乏工程完整度

## 执行原则

这 12 周最重要的不是“学了多少概念”，而是每周都能稳定产出：

- 可以运行的代码
- 可以展示的项目
- 可以写进简历的成果
- 可以在面试中讲清楚的实践经验

只要坚持按周推进、持续产出，这个学习计划就能真正帮助你在 3 个月后具备投递 AI Agent 应用开发实习的基础实力。
